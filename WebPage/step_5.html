Step 5. Develop preliminary network design



Up to this point, you already have accomplished various essential milestones in your journey to implement your fixed-wireless access (FWA) network. For instance, you have defined the type of services and the spectrum that will be used by your network to offer these services.

Moreover, the network architecture has been defined, which specifies the technologies to be used in your network. The next step in your journey is to develop the preliminary network design

The preliminary network design — commonly named as high-level design or HLD — defines the technology solutions to be implemented on each of the network sites that are part of the Network.

This is an optional step since, in most cases, consultants or vendors will develop the preliminary or high-level design for your Network. In addition, it requires an additional level of effort and more advanced technical skills than the previous steps.

In case you decide to go ahead, this step provides step-by-step instructions, tools, and examples to develop your own design.


5.1 Network Design Essentials

This section presents a general overview of the baseline concepts that you'll require to develop the preliminary network design.

5.1.1 End-to-end View

Figure 5.1 displays the generic network architecture of a fixed wireless access (FWA) network, which provides broadband services to fixed locations such as homes or business. You can find more details on the architectural elements in step 4.

Figure 5.1 — Generic architecture of a fixed wireless access network.

The principal elements that need to be defined as part of the network design process are:

  • Radio Access Network (RAN): Number and location of the required RAN site to cover the target polygons. Additionally, the definition of the equipment configuration to be implemented on each of RAN site.
  • Backhaul network: Backhaul technology to provide connectivity to each access site, considering the traffic generated by the expected subscribers to be served.
  • Core Site(s): Core site's location according to the solution defined in the Network Architecture.

5.1.2 RAN Design Considerations

The main objective of the RAN design is to estimate the appropriate number and location of the sites required to cover the target polygons. A couple of analyses must be performed in parallel to achieve this, as shown in Figure 5.2.

Figure 5.2 — Estimation of the required number of RAN sites

The coverage analysis determines the area that can be covered by each RAN site based on the propagation mechanisms on the network signal. On the other hand, the capacity analysis examines the traffic that each site needs to carry based on the expected subscribers.

The combination of the results from both analyses will determine the minimum number of RAN sites required to comply with the coverage and capacity requirements. Both analyses are included as part of the RAN Dimensioning Widget. The following sections provide a closer look at these analyses.


5.1.2.1 How much area can be covered by a RAN Site?

The area that a RAN site can cover depends on the radio link budget, which estimates the received signal level at the customer-premises equipment CPE from the base station.

The signal transmitted by the base station is subject to multiple losses and gains throughout its path up to the CPE location, such as propagation and interference loss and antenna gains. The radio link budget considers these losses and gains to estimate the maximum distance at which an appropriate service can be offered. Figure 5.3 displays a representation of the losses and gains affecting the radio signal in its path to the CPE location.

Figure 5.3 — Radio link budget representation

Based on the radio link budget results, the maximum coverage radius of each of the RAN sites can be calculated. This analysis is included as part of the RAN Dimensioning Widget.


5.1.2.2 How many users can be supported by a RAN Site?

The number of users that a RAN site can support depends on the radio equipment limitations. These limitations are primary determined by:

  • The number of sectors on each RAN site, which primary determines the number of radio equipment deployed on each site.
  • The maximum number of simultaneous users that can be supported by the radio equipment.
  • The maximum amount of traffic that can be supported by the radio equipment.

These factors must be considered when defining the maximum number of subscribers supported by each RAN site. This analysis is included as part of the RAN Dimensioning Widget.


5.1.3 Backhaul Design Considerations

This section describes the different backhaul technologies used to provide connectivity to the RAN sites and their primary design considerations. The principal technologies used in backhaul links are: fiber optic, microwave and satellite. You can find more details about these technologies in step 4.

5.1.3.1 Microwave Technology

Microwave backhaul uses antennas that are placed on towers or other tall structures at sufficient height to provide a direct, unobstructed line of sight (LOS) path between the transmitter and the receiver. In the case of microwave links, there are two main limitations to determine if they are feasible: the distance between the nodes and the LOS analysis.

The frequency used by the link primarily determines the maximum distances supported by a microwave link. In general, higher frequencies allow less distance between transmitter and receiver while increasing the link capacity. Table 5.1 includes a summary of the microwave links' characteristics.

MW Link Frequency

Characteristics

Frequencies below 15GHz

- Suitable for links up to 30km.
- In the case of unlicensed bands (i.e., 5 GHz) distance may be reduced due to interference.

Frequencies between 15GHz and 60GHz

- Suitable for links up to 10 km (depending on the specific frequency).
- Used for high-capacity links.

Table 5.1 — MW link characteristics based on their frequency

On the other hand, a microwave LOS analysis determines if any relevant obstacles may block the path from the transmitter to the receiver.

5.1.3.2 Fiber Optic Technology

Fiber optic is a wired backhaul technology with a significant, inherent bandwidth-carrying ability. But despite its operational capacity, its main limitation is its cost and deployment logistics.

The fiber optic deployment cost is mainly determined by the amount of fiber required to be deployed between the two sites that need to be connected. Therefore, fiber optic will be feasible when the amount of necessary fiber optic does not surpass an established threshold, which depends on the fiber deployment cost in the TL.

Furthermore, an additional condition to deploy fiber optic is the existence of a route connecting the sites that can be used to deploy the fiber optic cable. This route may include paved rural or urban roads, electrical grid, posts, or —more likely— some combination thereof.

5.1.3.3 Satellite Technology

Satellite technology is deployed in fringe network areas, usually in remote locations, where fiber optic and microwave technologies are not feasible.

Satellite link feasibility is primarily determined by the satellite footprint, which indicates the ground area covered by the satellite. The majority of commercial satellites provide their footprint in respective documentation, or you can access it from satellite footprint databases (e.g., Satbeams).

The cost of satellite terminal equipment located at the base station is comparable to microwave equipment. However, the OPEX burden generated by the satellite service fee can impact the business case.

5.1.4 Core Design Considerations

The core site is the principal aggregation point for all the network traffic and acts as the main connection point towards the internet.

In general terms, the core site is used to host the following elements:

  • Aggregation equipment (i.e., aggregation router and switch) use to concentrate the incoming network traffic and provide the connection towards the internet.
  • Centralized Evolved Packet Core (EPC) equipment in LTE scenario.
  • Systems' on-premise hardware used to manage the network operation.

During the design process, the location of the core site must be defined. Additionally, you can find more details regarding the elements included in the core site in step 4.

  1. Try to minimize the number of Greenfield sites due to their high cost of implementation.

    5.2.6 Task 6. Assess Backhaul technologies

    Once all the RAN sites have been defined, the next task is to provide connectivity for each of them. To this end, an assessment of each of the backhaul technologies (fiber optic, microwave and satellite) must be performed to validate their deployment feasibility. The following sections describe the procedures to assess the feasibility for each technology.

    Use the LOS Evaluation Tool to assess the Microwave Feasibility

    The LOS Evaluation Tool used on this module is airLink, which is a web-based LOS Evaluator to verify and validate LOS in MW Links. To assess if microwave is feasible to connect two sites, follow the instructions below:

    1. Locate the origin and destination nodes in the LOS Evaluation Tool.
    2. Introduce the tower height information for each of the sites. As a margin, each height is typically 2m below actual to account for space to implement the MW antennas.
    3. Use the options provided by the LOS Evaluation Tool to measure the distance between the sites, which represents the microwave link length.
    4. Select the frequency to be used based on the link length, using the values of as a reference.
    5. Evaluate the LOS using the options provided by the LOS Evaluation Tool.
    6. Verify that the path between transmitter and receiver is obstacle free (LOS is validated when there are none). Figure 5.34 depicts profiles generated for the example, where two analyses are performed. In this case, LOS is only validated for link B.

    Figure 5.34 — LOS validation using the LOS Evaluation Tool

    Use the GIS Tool to assess the Fiber Optic Feasibility

    To assess if fiber optic is feasible to connect two sites, follow the instructions below:

    1. Locate the origin and destination nodes in the GIS Tool.
    2. Trace a fiber route linking the origin and destination node using the options provided by the GIS Tool. This route shall follow main roads or any path between the nodes that can be used to deployed fiber optic, as shown in Figure 5.35.

  • Figure 5.35 — Example of fiber path design

    1. Using the GIS tool options, measure the length of the fiber route. This measurement represents the optical length of the fiber route.
    2. If the fiber route's optical length is below the threshold, fiber optic is a feasible technology to be used to connect these nodes.

    NOTE: If the fiber optic cost is not available, a good approximation is to consider 1km as the threshold value.

    Use the GIS Tool to assess the Satellite Feasibility

    To assess if satellite is feasible to provide connectivity in one location, follow the instructions below:

    1. Locate the origin node in the GIS Tool.
    2. Determine whether the location is inside the polygon defined by the satellite footprint.
    3. If the location is inside the satellite footprint, satellite is a feasible technology to be used to connect this node.

    5.2.7 Task 7. Define Backhaul technology and configuration

    At this point, you already know how to evaluate the feasibility of each backhaul technology. The next task in the process design is to select the appropriate one to provide connectivity for each RAN site. To do this, follow the instructions below for each of the RAN sites:

    2. Find the closest element node that is marked as backhaul site. This is called the next-hop site.

      1. First, evaluate the feasibility of Microwave technology.
      2. Second, evaluate the feasibility of Fiber Optic technology.
      3. Finally, evaluate the feasibility of Satellite technology.
    4. Use the HLD Template to record the information related to the selected backhaul network.

    5.3 Next steps

    Once you have developed your preliminary network design, there are some options to continue with your journey. These options are examined below:

    Support in the RFx Development (Step 6)

    The preliminary network design defines the technology solutions to be implemented on each site that compose the network. This includes the Bill of Quantities (BoQ) that determines the equipment count required to deploy the network. This information can be used to support the creation of an RFP addressed to vendors capable of providing the required network equipment and the related implementation services.

    Inform business plan (Step 7)

    Similar to the previous point, the preliminary network design information can be exported to be considered in the Financial Model. This information will render a more accurate view of the total cost to deploy the tribal land network.

    Interact with the network architecture (Step 4)

    Finally, you must consider that the network architecture can be modified based on the preliminary network design. The network architecture can include modifications to improve the network performance or a better adaption to your scenario.

    4 Next steps in the journey

    The network architecture module is linked with the following modules in the Tribal Land journey: Network Planning & Design and RFx & Vendor Assessment. These steps are shown in Figure 27 as steps 5 and 6 respectively of the Tribal Land Journey.

    • Network Planning & Design module

    The defined network architecture will serve as a guideline during the design process. Network designer must consider the resultant architecture as a starting point of its design. Main points to be considered are:

    1. RAN technology
    2. CPE option
    3. Spectrum band
    4. Transport technology
    5. Core architecture
    6. VLANs and IP addressing

    Keep in mind that the defined network architecture can be modified after the network planning & design process has been made. Network designer may conclude that modifications to certain network segment(s) could mean an improvement in terms of performance or improve the performance of the of the overall network or a better adaption to the Tribal Land scenario.

    • RFx & Vendor Assessment module

    An important part of the architecture wizard report is the section containing the Engineering Guidelines. These guidelines summarize required equipment specifications to support the recommended network architecture.

    These equipment requirements must be considered during the RFx process for the creation of the RFP and for vendor evaluation. Equipment requirements cover all segments of the FWA network: RAN, transport, core, IP, and systems.
